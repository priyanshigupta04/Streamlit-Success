🎓 Automated NOC Generation System

Implementation Status: COMPLETED

Project: No Objection Certificate (NOC) Automation

Automated generation of professional No Objection Certificates from template and student/job data, with full approval workflow integration.

📦 What Was Implemented

1. Template Engine

File Purpose Status
backend/templates/noc.hbs Handlebars template for NOC document ✓

2. Services Layer

File Function Status
backend/services/documentGenerator.js Core NOC generation logic ✓

Key Functions:

  • generateNoc(studentId, jobId) - Main generation function
  • validateNocRequirements(studentId, jobId) - Validates required data
  • getNocPreview() - Returns template preview HTML
  • fetchNocData(studentId, jobId) - Fetches student and job data
  • formatDataForTemplate(student, jobData) - Formats data for rendering
  • htmlToPdf(html, fileName) - Converts HTML to PDF
  • uploadToCloudinary(pdfBuffer, fileName, studentId) - Uploads PDF

3. API Endpoints

Endpoint Method Auth Required Purpose
/api/documents/ POST Student Request a document (NOC, Bonafide, etc.)
/api/documents/mine GET Student Get student's documents
/api/documents/pending GET Mentor/HOD/Dean Get documents pending approval
/api/documents/:id/approve PUT Mentor/HOD/Dean Approve/reject & auto-generate PDF on dean approval
/api/documents/all GET Placement Cell View all documents
/api/documents/noc/preview GET None Preview NOC template
/api/documents/noc/validate POST Student Validate NOC requirements before request
/api/documents/:id/regenerate POST Student Regenerate PDF if data changes

4. Database Updates

DocumentRequest Model: Added fields for tracking generation:

generatedDocUrl    String    // Cloudinary PDF URL
generatedAt        Date      // When PDF was created
signatureUrl       String    // Dean's digital signature
signedAt           Date      // When signed
issueDate          Date      // Official issue date
expiryDate         Date      // Validity expiration
documentVersion    Number    // Track re-generations

metadata: {
  studentName      String
  enrollmentNo     String
  branch           String
  year             Number
  dateOfGeneration Date
}

🔄 NOC Generation Workflow

Step-by-Step Process:

  1. Student Request: Student requests NOC with job details 
    POST /api/documents/
{
  "type": "noc",
  "reason": "Need NOC for internship",
  "jobId": "ObjectId"
}
  2. Mentor Approval: Mentor reviews and approves 
    PUT /api/documents/:id/approve
{
  "decision": "approved",
  "note": "Looks good"
}
    overallStatus → mentor_approved
  3. HOD Approval: HOD reviews mentor's approval 
    PUT /api/documents/:id/approve
{
  "decision": "approved",
  "note": "Approved"
}
    overallStatus → hod_approved
  4. Dean Approval + PDF Generation: Dean approves and PDF is automatically generated 
    PUT /api/documents/:id/approve
{
  "decision": "approved",
  "note": "Approved"
}
    • ✓ overallStatus → issued
    • ✓ NOC HTML generated from template
    • ✓ HTML converted to PDF
    • ✓ PDF uploaded to Cloudinary
    • ✓ generatedDocUrl saved in DB
    • ✓ metadata populated from student data
    • ✓ generatedAt timestamp recorded
  5. Student Retrieves PDF: Student downloads from NOC 
    GET /api/documents/mine
[
  {
    _id: "...",
    type: "noc",
    overallStatus: "issued",
    generatedDocUrl: "https://cloudinary.url/...",
    generatedAt: "2025-03-13T...",
    metadata: {...}
  }
]

Data Mapping (Template Variables)

Student Data Source (User Model):
  name           → studentName
  enrollmentNo   → enrollmentNo
  branch         → branch
  year           → year
  cgpa           → cgpa
  graduationYear → academicYear (calculated)
  mentorId       → mentorName

Job Data Source (Job Model):
  company        → companyName
  title          → jobTitle
  location       → jobLocation
  stipend        → stipend

Auto-Generated:
  issueDate      → Current date
  documentId     → From studentId last 8 chars
  validityPeriod → "1 year"

📝 API Usage Examples

1. Request NOC

curl -X POST http://localhost:5000/api/documents/ \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "type": "noc",
    "reason": "Need NOC for internship at TechCorp",
    "jobId": "64a5f8c3d9e1a2b3c4d5e6f7"
  }'

2. Validate NOC Requirements

curl -X POST http://localhost:5000/api/documents/noc/validate \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "studentId": "64a5f8c3d9e1a2b3c4d5e6f7",
    "jobId": "64a5f8c3d9e1a2b3c4d5e6f8"
  }'

Response:
{
  "valid": true,
  "errors": [],
  "message": "NOC can be requested"
}

3. Check NOC Status

curl -X GET http://localhost:5000/api/documents/mine \
  -H "Authorization: Bearer YOUR_TOKEN"

Response:
{
  "documents": [
    {
      "_id": "...",
      "type": "noc",
      "overallStatus": "issued",
      "generatedDocUrl": "https://cloudinary.url/noc_file.pdf",
      "generatedAt": "2025-03-13T10:30:00Z",
      "metadata": {
        "studentName": "John Doe",
        "enrollmentNo": "ENG2021001",
        "branch": "CSE",
        "year": 4
      }
    }
  ]
}

4. Approve NOC (Triggers Generation)

curl -X PUT http://localhost:5000/api/documents/64a5f8c3d9e1a2b3c4d5e6f7/approve \
  -H "Authorization: Bearer DEAN_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "decision": "approved",
    "note": "Approved for placement"
  }'

Response:
{
  "document": {
    "type": "noc",
    "overallStatus": "issued",
    "generatedDocUrl": "https://cloudinary.url/NOC_ENG2021001_1710326400000.pdf",
    "generatedAt": "2025-03-13T10:30:00Z",
    "documentVersion": 1,
    "metadata": {...}
  }
}
